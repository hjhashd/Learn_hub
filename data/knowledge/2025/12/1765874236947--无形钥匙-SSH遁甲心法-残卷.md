# 📜 《无形钥匙·SSH遁甲心法》残卷

> *"凡人每次登录远程道场（服务器）皆需口诵密咒（密码），实乃浪费灵力。本座今传你一卷古法，让你与远程道场建立**血契**，从此自由出入，再无阻碍！"*

---

## 🌩️ 渡劫缘起（背景痛点）

本座原本想随时穿梭于多个远程道场（服务器）之间，却每次都要输入繁琐的密码咒语，烦不胜烦！  
尤其是在 VS Code 连接远程服务器时，那破弹窗天天跳出来要密码，简直像渡劫失败的心魔反噬。

更蠢的是，曾想直接在 SSH config 里写个 `Password` 字段——结果被天道打脸：**SSH 根本不支持这种明文存密的骚操作！**（为了防止你被盗号后密码直接泄露）

于是，必须祭出行业标准秘术：**SSH 密钥对免密登录**！

---

## 🔮 破阵心法（核心原理）

### 壹·钥匙与锁的玄学

传统登录方式：
```
你每次敲门 → 守门人问："口令？" → 你输密码 → 放行
```

SSH 密钥登录（本座之法）：
```
首次拜山：你给守门人刻一把你的专属印记（公钥）
以后每次：你亮出信物（私钥）→ 守门人核对印记 → 直接放行！
```

**核心机制**：
- **私钥（id_rsa）**：藏在你的肉身（本地电脑）里，绝不外传，相当于你的"灵魂指纹"。
- **公钥（id_rsa.pub）**：可以到处发，放到服务器的白名单（`~/.ssh/authorized_keys`）里。
- **验证过程**：服务器用公钥加密挑战 → 你用私钥解密 → 服务器确认"是本人" → 免密进入！

### 贰·为何 Windows 凡人总遭天劫？

你会发现：
- `ssh-copy-id` 命令报错 → 因为这是 Linux 的法术，Windows PowerShell 不认识！
- `nano` 编辑器打不开 → 因为你在本地（Windows）施展了远程（Linux）的咒语！

**正确姿势**：  
- 在 Windows 用 PowerShell 专属命令：`type` + `ssh` 组合技
- 在 Linux 服务器内才能用 `nano` / `vim` 等原生法器

## 👹 禁术·施法步骤

### 第一式：锻造无形钥匙（生成密钥对）

**在本地 Windows PowerShell 施展**：

```powershell
# 😈 吾以灵气凝聚，铸造专属印记！
ssh-keygen -t rsa
```

**操作要诀**：
- 一路狂按 `Enter`（回车），不要输入任何 passphrase（密码保护）
- 若已有密钥想覆盖，按 `y` 即可

**生成产物**：
- `C:\Users\你的用户名\.ssh\id_rsa` ← 私钥（灵魂本体，严禁外传！）
- `C:\Users\你的用户名\.ssh\id_rsa.pub` ← 公钥（可随意复制给服务器）

---

### 第二式：传送印记（将公钥注入服务器白名单）

#### 💀 情况A：标准22端口服务器

**目标**：`cqj@192.168.3.10`（默认端口22）

```powershell
# 🔥 窃取天机·一键传功大法！
type $env:USERPROFILE\.ssh\id_rsa.pub | ssh cqj@192.168.3.10 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

**命令解析**：
- `type $env:USERPROFILE\.ssh\id_rsa.pub`：读取公钥内容
- `|`：传递给下一个命令（管道术）
- `ssh cqj@192.168.3.10 "..."`：通过 SSH 在服务器上执行命令
- `mkdir -p ~/.ssh`：确保目录存在（若已存在不报错）
- `cat >> ~/.ssh/authorized_keys`：将公钥追加到白名单

**运行现象**：
- 系统会要求输入**最后一次密码**（以后再也不用输了）
- 输入时屏幕无显示（盲输），属正常现象
- 无报错即成功！

---

#### 💀 情况B：非标端口服务器

**目标**：`cqj@127.0.0.1`（端口11123，VPN 映射）

```powershell
# ⚡ 针对非标端口，需加 -p 参数指定门牌号！
type $env:USERPROFILE\.ssh\id_rsa.pub | ssh -p 11123 cqj@127.0.0.1 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

**关键点**：`-p 11123` 指定端口，否则默认走22端口会找不到门。

---

#### 💀 情况C：云服务器（如腾讯云）

**目标**：`ubuntu@129.226.202.159`

```powershell
# 🌩️ 针对云端道场，亦用此法！
type $env:USERPROFILE\.ssh\id_rsa.pub | ssh ubuntu@129.226.202.159 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

## 🗡️ 进阶邪术

### 壹·转生术：切换到 Root 身份登录

**场景**：发现普通用户权限不够，想直接用 `root` 登录。

#### Step 1：在服务器开启 Root 登录权限

先用普通用户（如 `ubuntu`）登录服务器，然后：

```bash
# 🔓 解除天道对 Root 的封印
sudo vim /etc/ssh/sshd_config

# 找到这一行（按 / 搜索 "PermitRootLogin"）
PermitRootLogin yes           # 改成 yes

# 确保密码登录也开启（首次需要用密码传公钥）
PasswordAuthentication yes    # 改成 yes
```

**保存退出**：
- 按 `Esc` → 输入 `:wq` → 回车

**重启 SSH 服务使配置生效**：
```bash
sudo systemctl restart sshd
# 或
sudo service ssh restart
```

#### Step 2：给 Root 也配上免密

回到本地 PowerShell：

```powershell
# ⚠️ 注意：Root 的白名单和普通用户是分开的！
type $env:USERPROFILE\.ssh\id_rsa.pub | ssh root@129.226.202.159 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

输入一次 root 密码后，以后 `ssh root@129.226.202.159` 就再也不用密码了！

### 贰·困境解救：被 Vim 困住了怎么办？

**经典灾难现场**：不小心进了 Vim，按什么都没反应，屏幕一直闪光标……

**逃生口诀**：

```bash
# 1. 狂按 Esc 键 3 次（退出编辑模式）

# 2. 输入以下任一命令：

:wq   # 保存并退出（w=write, q=quit）
:q!   # 强制退出不保存（! 表示强制）
```

**原理科普**：
- Vim 有两个世界："编辑模式"（可打字）和"命令模式"（可执行操作）
- 按 `i` 进入编辑模式，按 `Esc` 回到命令模式
- 只有在命令模式下输入 `:` 开头的指令才会生效

### 叁·多服务器批量配置

如果你有一堆服务器要配免密，可以写个循环脚本：

```powershell
# 😈 吾以阵法覆盖诸界！
$servers = @(
    "cqj@192.168.3.10",
    "ubuntu@129.226.202.159",
    "root@123.456.78.90"
)

foreach ($server in $servers) {
    Write-Host "🔥 正在对 $server 施展传功大法..." -ForegroundColor Cyan
    type $env:USERPROFILE\.ssh\id_rsa.pub | ssh $server "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
}

Write-Host "✅ 所有道场已建立血契！" -ForegroundColor Green
```

## ⚠️ 防走火入魔指南

| 禁忌行为 | 后果 | 解决之道 |
|---------|------|---------|
| 把 `id_rsa` 私钥传给别人 | 相当于把身份证给了陌生人，他能冒充你登录所有服务器！ | **严禁外传！**只能传公钥 `id_rsa.pub` |
| 在本地 PowerShell 运行 `nano` | 报错：找不到命令 | 先 `ssh` 登录服务器再用 Linux 命令 |
| 在 config 里写 `Password` 字段 | 根本不支持，白费力气 | 用密钥登录才是正道 |
| 修改 `sshd_config` 后不重启服务 | 配置不生效，白改 | 记得 `sudo systemctl restart sshd` |
| Root 和普通用户共用白名单 | 不存在的！两者的 `~/.ssh/` 是独立的 | 分别传一次公钥 |
| 端口不是22却不加 `-p` 参数 | 连不上，报 Connection refused | 非标端口必须加 `-p 端口号` |

## 📖 常见天劫及破解之法

### 天劫一：Permission denied (publickey)

**症状**：配置完免密后还是被拒绝，提示权限错误。

**病因**：
1. 服务器上 `~/.ssh/` 目录或 `authorized_keys` 文件权限不对
2. SELinux 或云服务商的安全策略阻拦

**解药**：

```bash
# 登录服务器，修正权限（必须严格！）
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

# 确保文件属主是当前用户
ls -la ~/.ssh/
# 若显示 root 拥有而你是 ubuntu，需执行：
sudo chown -R ubuntu:ubuntu ~/.ssh
```

---

### 天劫二：WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!

**症状**：重装过服务器系统后，再连接时报恐怖的红字警告。

**病因**：服务器的"指纹"变了（重装后 SSH Host Key 重新生成），但你本地缓存的还是旧指纹。

**解药**：

```powershell
# 删除本地缓存的旧指纹
ssh-keygen -R 服务器IP

# 例如：
ssh-keygen -R 192.168.3.10
```

然后重新连接，输入 `yes` 接受新指纹即可。

---

### 天劫三：ssh-copy-id 不存在

**症状**：在 Windows PowerShell 运行 `ssh-copy-id` 报错。

**病因**：这是 Linux 原生命令，Windows 默认没有。

**解药**：用本座的 PowerShell 版替代命令（见上文第二式）。

## 🌟 彩蛋：一键生成 SSH Config

懒得每次都输 `ssh user@host -p port`？写个配置文件让 VS Code 自动识别！

**编辑文件**：`C:\Users\你的用户名\.ssh\config`

```ssh-config
# 📝 赛博邪修の传送阵目录

# 内网服务器
Host 10server
    HostName 192.168.3.10
    User cqj
    Port 22

# VPN 映射端口
Host vpn10
    HostName 127.0.0.1
    User cqj
    Port 11123

# 腾讯云服务器
Host txcloud
    HostName 129.226.202.159
    User ubuntu

# 阿里云 Root 专用
Host alicloud-root
    HostName 123.45.67.89
    User root
    Port 22
```

**使用姿势**：

```powershell
# 直接用别名连接！
ssh 10server
ssh txcloud
ssh alicloud-root
```

VS Code 也会自动读取这个配置，远程资源管理器里直接显示这些名字！
