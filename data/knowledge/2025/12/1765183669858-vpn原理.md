## 一、VPN 基础原理[[还有份搞怪版本：vpn飞升版]]

### 1.1 核心概念
VPN（虚拟专用网络）是在公用网络上建立的**加密专用隧道**。

**工作流程：**
1. **打包与加密**：客户端将请求封装并加密
2. **传输**：加密数据通过 ISP 传输到 VPN 服务器
3. **解密与转发**：VPN 服务器解密后以自己名义访问目标网站
4. **回传**：服务器将响应加密后返回客户端

### 1.2 VPN vs 普通代理
- **普通代理**：仅代理特定软件流量
- **VPN（隧道）**：在操作系统网络层工作，所有流量都通过加密隧道

## 二、云服务器选购指南

### 2.1 关键选择标准

**线路优先级（最重要）：**
1. **CN2 GIA**（推荐）：中国电信下一代承载网，晚高峰不堵车
2. **AS9929**：中国联通精品网，性价比高
3. 普通线路：延迟高，晚高峰易丢包

**地区推荐：**
- **香港**：延迟低（40ms），玩游戏首选，但带宽较贵
- **美国洛杉矶 + CN2 GIA**：性价比之王，延迟 150ms，带宽大
- **新加坡**：速度不错，性价比高于香港

### 2.2 国内云厂商对比

| 厂商 | 推荐指数 | 优势 | 劣势 |
|------|---------|------|------|
| 腾讯云 | ⭐⭐⭐⭐⭐ | 性价比高，香港线路优化好，新人特惠（99元/年） | 续费可能变贵 |

## 三、服务器环境配置

### 3.1 初始连接

**推荐工具：VS Code + Remote SSH**

```bash
# 安装 Remote - SSH 插件后添加主机
ssh ubuntu@<服务器IP>
```

### 3.2 基础配置

**1. 切换到 root 权限**
```bash
sudo -i
```

**2. 开启 BBR 加速（提升网络性能）**
```bash
echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p
```

**3. 验证 Docker**
```bash
docker ps
```

### 3.3 防火墙配置

必须在云服务商控制台开放端口：
- **2053**：管理面板端口
- **443**：VPN 数据传输端口（TCP/UDP）

## 四、部署 VPN 服务（X-UI）

### 4.1 使用 Docker Compose 部署

**创建配置文件：**
```bash
mkdir -p /root/x-ui && cd /root/x-ui
```

**docker-compose.yml：**
```yaml
version: '3'
services:
  x-ui:
    image: ghcr.io/mhsanaei/3x-ui:latest
    container_name: x-ui
    volumes:
      - ./db/:/etc/x-ui/
      - ./cert/:/root/cert/
    tmpfs:
      - /tmp
    restart: always
    network_mode: host
    environment:
      XRAY_VMESS_AEAD_FORCED: "false"
```

**启动服务：**
```bash
docker compose up -d
```

### 4.2 配置 VLESS-Reality 节点

**访问管理面板：** `http://<服务器IP>:2053`
- 默认账号：admin
- 默认密码：admin
- **首次登录后立即修改密码**

**创建入站配置：**
1. **基本设置**
   - 协议：`vless`
   - 端口：`443`

2. **客户端设置**
   - 流控：`xtls-rprx-vision`

3. **传输配置**
   - 传输协议：`tcp`
   - 安全性：`reality`
   - uTLS：`chrome`
   - Dest：`www.microsoft.com:443`
   - SNI：`www.microsoft.com`
   - Private Key：点击生成
   - ShortId：点击生成

### 4.3 多设备管理

**为不同设备分配标识：**
```
电脑 - email: pc
iPad - email: ipad
手机 - email: phone
```

这样可以在日志中区分不同设备的访问记录。

## 五、客户端配置

### 5.1 Windows - v2rayN

1. 下载 v2rayN-With-Core.zip
2. 导入节点：复制 `vless://` 链接后 Ctrl+V
3. 设为活动服务器
4. 系统代理 → 自动配置系统代理
5. 路由模式：选择"绕过大陆"

### 5.2 iOS - Shadowrocket

1. 扫描管理面板生成的二维码
2. 允许添加 VPN 配置
3. 开启连接开关

### 5.3 Android - v2rayNG

下载文件选择：
- **首选**：`v2rayNG_*_arm64-v8a.apk`（适用于现代手机）
- **备选**：`v2rayNG_*_universal.apk`（通用版）

导入方式：扫描二维码或从剪贴板导入

## 六、日志与监控

### 6.1 开启访问日志

**在 X-UI 面板 → Xray 设置：**
- 访问日志路径：`/etc/x-ui/access.log`
- 错误日志路径：`/etc/x-ui/error.log`
- 日志级别：`info`

### 6.2 查看日志

**实时监控：**
```bash
tail -f /root/x-ui/db/access.log
```

**筛选特定设备：**
```bash
grep "email: ipad" /root/x-ui/db/access.log
```

**查找访问记录：**
```bash
grep "youtube" /root/x-ui/db/access.log
```

**日志格式示例：**
```
2025/11/27 15:30:00 127.0.0.1:54321 accepted tcp:www.google.com:443 [vless-inbound] email: ipad
```

## 七、安全最佳实践

### 7.1 服务器安全

1. **修改默认密码**：管理面板和 SSH 都要改
2. **复杂密码策略**：大小写+数字+符号，16位以上
3. **考虑密钥登录**：禁用 SSH 密码登录（进阶）

### 7.2 使用规范

**✅ 安全用途：**
- 查阅技术资料
- 观看视频学习
- 正常娱乐

**❌ 高风险行为：**
- 发表违法言论
- 访问极端敏感网站
- 涉及黄赌毒等违法内容
- 分享给大量他人使用

### 7.3 技术细节

**加密保护：**
- ISP 只能看到你连接了服务器，无法看到传输内容
- Reality 协议伪装成正常 HTTPS 流量
- 腾讯云无法轻易解密实时流量内容

**元数据透明：**
- 服务商知道连接时间和流量大小
- 可推测出是代理服务器（但只要不触发风控通常不管）

**HTTPS 限制：**
- 日志只能看到域名（如 `youtube.com`）
- 看不到具体访问路径（如具体视频 ID）

## 八、常见问题排查

### 8.1 无法连接

**检查清单：**
1. ✓ 防火墙是否开放 443 端口
2. ✓ 客户端配置是否为最新（修改后需重新导入）
3. ✓ Dest 和 SNI 是否一致
4. ✓ 是否有其他服务占用 443 端口（如 Nginx）

**端口占用检查：**
```bash
netstat -nlp | grep 443
```

### 8.2 数据中心 IP 问题

**影响：**
- 对大部分网站无影响（Google、YouTube、GitHub 等）
- Netflix 等流媒体可能限制
- 防火墙封锁主要看流量特征，不是 IP 类型

**解决方案：**
- Reality 协议伪装流量特征
- 低调使用，避免大流量
